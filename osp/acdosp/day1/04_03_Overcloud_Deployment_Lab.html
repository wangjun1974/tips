Overcloud Deployment Lab

Goals

  • Deploy the overcloud

  • Review the overcloud deployment

1. Deploy Overcloud

In this section, you deploy the overcloud.

Use screen or tmux to safeguard the terminal session.

  1. As the stack user, create a script containing the overcloud deploy command with all of the necessary options:

    Sample deploy.sh Script
    #!/bin/bash
THT=/usr/share/openstack-tripleo-heat-templates/
CNF=~/templates/

source ~/stackrc
openstack overcloud deploy --templates $THT \
-r $CNF/roles_data.yaml \
-n $CNF/network_data.yaml \
-e $THT/environments/network-isolation.yaml \
-e $CNF/environments/network-environment.yaml \
-e $CNF/environments/storage-environment.yaml \
-e $CNF/environments/net-bond-with-vlans.yaml \
-e ~/containers-prepare-parameter.yaml \
-e $CNF/node-info.yaml \
-e $CNF/fix-nova-reserved-host-memory.yaml
    File network-isolation.yaml is generated during deployment from jinja template. See https://access.redhat.com/solutions/3299211

  2. Make the script executable:

    (undercloud) [stack@undercloud ~]$ chmod 755 ~/deploy.sh

  3. Run the script from the undercloud node:

    (undercloud) [stack@undercloud ~]$ time ./deploy.sh

    • This initiates overcloud deployment according to the supplied environment files and templates.

      The deployment takes approximately an hour.

  4. Open a second SSH session to the undercloud node to monitor the overcloud deployment process using the openstack stack list command:

    [stack@undercloud ~]$ source ~/stackrc
(undercloud) [stack@undercloud ~]$ watch -n 30 "openstack stack list --nested"

    Once the stack is created completily, Ansible playbooks will configure the servers on the Overcloud.

  5. After successful deployment of the overcloud, locate the following files under the stack user’s home directory:

    File NameDescription

    overcloudrc

    Environment variables allowing access to overcloud

    tempest-deployer-input.conf

    Tempest configuration file used to validate installation
